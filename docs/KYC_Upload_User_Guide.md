# EBIX Remittance Maker - KYC Upload User Guide

## Overview

The KYC (Know Your Customer) Upload feature allows EBIX agents to upload and manage required identity documents for remittance transactions. This ensures compliance with regulatory requirements and helps prevent fraud.

## Getting Started

### Accessing KYC Upload
1. Log in to the EBIX Web Agent Portal
2. Navigate to the "Transaction" section
3. Click on the "KYC Upload" tab
4. View the list of transactions requiring document uploads

## KYC Upload Process

### Step 1: Review Transaction List

The main screen displays a table of transactions that require KYC document uploads.

#### Table Information:
- **Transaction ID**: Unique identifier for each transaction
- **Applicant Name**: Name of the person sending money
- **Amount**: Transaction amount and currency
- **Status**: Current KYC verification status
- **Created Date**: When the transaction was created
- **Actions**: Available actions for each transaction

#### Status Types:
- **Pending**: Documents need to be uploaded
- **Under Review**: Documents submitted, awaiting verification
- **Approved**: Documents verified successfully
- **Rejected**: Documents need revision (see rejection reasons)

### Step 2: Upload Documents

#### Initiating Upload
1. Find the transaction in the list
2. Click the "Upload" button in the Actions column
3. The upload form will open for that specific transaction

#### Transaction Information
The form displays read-only information about the transaction:
- **Company Reference Number**: EBIX's internal reference
- **Agent Reference Number**: Agent's reference number
- **Applicant Name**: Name of the customer

### Step 3: Document Requirements

#### Dynamic Document List
The system automatically determines required documents based on:
- **Transaction Purpose**: Different purposes require different documents
- **Transaction Amount**: Higher amounts may require additional verification
- **Destination Country**: Some countries have specific document requirements

#### Common Document Types:
- **Passport**: Indian passport (front and back)
- **PAN Card**: Permanent Account Number
- **Aadhaar Card**: National identity card
- **Bank Statements**: Proof of funds source
- **Address Proof**: Utility bills, rental agreements
- **Education Documents**: For education-related transactions
- **Medical Documents**: For medical treatment transactions

#### Document Specifications:
- **Format**: PDF, JPG, JPEG, PNG
- **Size Limit**: Maximum 5MB per document
- **Quality**: Documents must be clear and readable
- **Validity**: Documents should be current and not expired

### Step 4: Upload Process

#### Single-Sided Documents
For documents that require only one side (PAN Card, Aadhaar):
1. Click "Choose File" or drag and drop the file
2. Select the document file from your computer
3. The file will upload automatically
4. Green checkmark indicates successful upload

#### Double-Sided Documents
For documents requiring both sides (Passport):
1. **Front Side**: Upload the front page
2. **Back Side**: Upload the back page (if applicable)
3. Both sides must be uploaded for complete verification

#### Upload Status Indicators:
- **Uploading**: Progress bar shows upload progress
- **Success**: Green checkmark and success message
- **Error**: Red error message with details

### Step 5: Handle Rejections

#### Understanding Rejections
If documents are rejected, the system will show:
- **Rejection Reason**: Why the document was not accepted
- **Rejected By**: Who reviewed and rejected the document
- **Rejection Date**: When the rejection occurred
- **Remarks**: Additional comments from the reviewer

#### Common Rejection Reasons:
- **Unclear Image**: Document not readable
- **Expired Document**: Document validity expired
- **Incomplete Information**: Missing required details
- **Mismatched Information**: Document doesn't match transaction details
- **Wrong Document Type**: Incorrect document uploaded

#### Re-upload Process:
1. Click "Re-upload" for rejected transactions
2. Review rejection reasons carefully
3. Upload corrected or additional documents
4. Address all rejection comments

### Step 6: Complete Upload

#### Review Before Submission
- Ensure all required documents are uploaded
- Check upload status for each document
- Verify file quality and readability

#### Submission
- Click "Submit" when all documents are uploaded
- System will validate all requirements
- Success message confirms submission

## Document Management

### Viewing Uploaded Documents
- Uploaded documents can be viewed in preview mode
- Click on document thumbnails to enlarge
- Download copies if needed

### Document History
- Track all upload attempts
- View previous versions
- See approval/rejection timeline

### Bulk Operations
- Upload multiple documents for one transaction
- Batch processing for efficiency

## Best Practices

### Document Preparation
- **Scan Quality**: Use high-resolution scanning (at least 300 DPI)
- **File Format**: Convert to PDF for better compression
- **File Naming**: Use clear, descriptive names
- **File Size**: Compress large files without losing quality

### Upload Tips
- **Internet Connection**: Ensure stable connection during upload
- **File Validation**: Check files before uploading
- **Backup Copies**: Keep copies of all uploaded documents
- **Organization**: Group related documents logically

### Compliance Guidelines
- **Customer Consent**: Ensure customer permission for document collection
- **Data Privacy**: Handle documents securely and confidentially
- **Regulatory Requirements**: Follow KYC and AML guidelines
- **Record Keeping**: Maintain proper documentation records

## Troubleshooting

#### Network Issues
- **Problem**: Upload interrupted
- **Solution**: Check internet connection and retry
- **Tip**: Use wired connection for large files

### Document Quality Issues

#### Blurry Images
- **Problem**: Document not readable
- **Solution**: Retake photo/scan with better quality
- **Tip**: Use good lighting and steady camera

#### Incomplete Documents
- **Problem**: Missing information
- **Solution**: Upload complete document or additional pages
- **Tip**: Check document requirements before scanning

### System Errors

#### Transaction Not Found
- **Problem**: Cannot access upload form
- **Solution**: Refresh page or contact support
- **Tip**: Note transaction ID for reference

#### Permission Denied
- **Problem**: Cannot upload documents
- **Solution**: Check user permissions or contact administrator
- **Tip**: Ensure you're logged in with correct credentials


### Customer Privacy
- Documents used only for verification purposes
- Not shared with third parties without consent
- Retained as per regulatory requirements

### Access Control
- Role-based access to KYC functions
- Audit trail for all document access
- Secure login required for all operations
