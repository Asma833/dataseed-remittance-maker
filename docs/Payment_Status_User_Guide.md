# EBIX Remittance Maker - Payment Status User Guide

## Overview

The Payment Status feature allows EBIX agents to monitor and manage payment processing for remittance transactions. This includes tracking payment statuses, uploading payment proofs, and managing transaction lifecycles.

## Getting Started

### Accessing Payment Status
1. Log in to the EBIX Web Agent Portal
2. Navigate to the "Transaction" section
3. Click on the "Payment Status" tab
4. View the comprehensive payment status dashboard

## Payment Status Dashboard

### Understanding the Payment Table

The main dashboard displays all transactions with their payment information in a comprehensive table format.

#### Key Information Displayed:

**Transaction Identifiers**
- **Ref. No**: EBIX's company reference number
- **Agent Ref. No**: Agent's reference number for the transaction
- **Order Date**: When the transaction was created
- **Expiry Date**: Deadline for payment completion

**Customer Information**
- **Applicant Name**: Name of the person sending money
- **Applicant PAN Number**: Customer's Permanent Account Number

**Transaction Details**
- **Transaction Type**: Type of remittance transaction
- **Purpose**: Reason for the transaction (Personal, Education, Medical, etc.)
- **FX Currency**: Foreign currency code
- **FX Amount**: Amount in foreign currency
- **Settlement Rate**: Exchange rate for settlement
- **Customer Rate**: Rate shown to customer
- **Transaction Amount**: Total amount in INR



#### Filter and Search
- **Search**: Use the search bar to find specific transactions by reference number, name, or PAN
- **Date Range Filter**: Filter transactions by order date using the date picker
- **Status Filters**: Filter by KYC status, payment status, or transaction type
- **Export**: Download the filtered data as CSV for reporting

#### Transaction Assessment
- Review all status indicators
- Check expiry dates to ensure timely processing
- Identify transactions requiring action

### Step 2: Upload Payment Proof

#### When to Upload Payment Screenshots
- When customer has made the payment
- For offline bank transfers
- As proof of payment completion
- Before transaction expiry

#### Upload Process
1. **Locate Transaction**: Find the transaction in the table
2. **Click Upload Button**: Click the upload icon in the "Payment Screenshot" column
3. **Select Payment Method**: Choose the payment method used (if applicable)
4. **Upload File**: Select and upload the payment screenshot or challan
5. **Submit**: Confirm the upload

#### File Requirements
- **Format**: JPG, PNG, PDF
- **Quality**: Clear, readable image
- **Content**: Must show payment details, amount, date, and transaction reference

### Step 3: Monitor Payment Progress

#### Track Status Changes
- **Real-time Updates**: Status changes reflect automatically
- **Email Notifications**: Receive alerts for status changes
- **Audit Trail**: Complete history of payment activities


## Transaction Management

### View Transaction Details

#### Accessing Full Details
1. Click the "View" button in the Actions column
2. Opens the complete transaction in view mode
3. Review all transaction information
4. Access related documents and history

#### Available Actions in View Mode
- **Download PDF**: Generate transaction summary
- **View Documents**: Access uploaded KYC documents
- **Payment History**: Review payment timeline
- **Status Updates**: Monitor progress

### Transaction Lifecycle

#### Complete Transaction Flow
1. **Transaction Created**: Initial setup by agent
2. **KYC Pending**: Documents required
3. **KYC Approved**: Documents verified
4. **Payment Pending**: Awaiting customer payment
5. **Payment Processing**: Payment verification
6. **Payment Completed**: Transaction ready for processing
7. **Funds Transferred**: Money sent to beneficiary

#### Time-sensitive Actions
- **KYC Upload**: Usually within 24-48 hours
- **Payment**: Must be completed before expiry date
- **Processing**: Typically 1-3 business days after payment

## Reporting and Analytics

### Export Capabilities
- **CSV Export**: Download filtered data for analysis
- **Date Range Reports**: Generate reports for specific periods
- **Status Reports**: Track conversion rates and processing times
